prima cosa -> leggere file json

il file json è immenso (15 GB)

serve un modo per leggerlo passo passo invece che tutto insieme
(manco entrerebbe in ram tutto insieme)

ogni tabella è in una riga del file

quindi è easy leggerlo passo passo

leggiamo una riga, e quindi una tabella, alla volta e con la libreria
gson possiamo convertirla in un oggetto java

in questo modo leggiamo una tabella, la indicizziamo, e passiamo
alla prossima fino a completare l'indice

adesso serve da fare:
 - "TablesHandler": classe che converte le tabelle in set di documenti,                 -giordano
                   nello specifico le colonne della tabella in documenti
                   deve anche creare le statistiche (ad esempio numero medio
                   di colonne, di righe, numero di tabelle totali, etc)
 - capire che analyzer usare
 - "IndexHandler": classe che gestisce l'indice, con un metodo che prende               -riccardo
                   i documenti e li aggiunge all'indice.
 - "QueryHandler": gli dai un "documento", ovvero una colonna di una tabella,           -simone
                   ti trova le tabelle che possono espandere la tabella, con
                   algoritmo merge list

 - testing: creare un piccolo dataset e testare su quello
                   testare su tables.json
 - presentazione


note: ci sono tabelle senza gli headers
    -> campi header di queste tabelle impostati "header"
